### 1. Требования безопасности к внешним системам
- Взаимодействие по HTTPS/TLS для защиты от перехвата данных 
- Хранение данных пользователей в зашифрованном виде 
- Регулярная ротация ключей шифрования  
- Возможность удаления устаревших данных 
- Наличие аудита критически важных событий
- Наличие аутентификации и авторизации по RBAC на сервере 

### 2. Протоколы аутентификации и авторизации
 Для аутентификации и авторизации подходит протокол OIDC так как основан на передаче токенов, без раскрытия личных данных о пользователе 

### 3. Механизм взаимодействия между системами
- От внешней платформы к приложению: Данные (уведомление о звонке) поступают в gateway → аутентификация (Keycloak с OIDC) → iot-device-adapter -> tenant-core-app → передача в мобильное приложение. 
- От приложения к внешней платформе: Команды (регистрация пользователя, открытие двери/шлагбаума) генерируются в мобильном приложении → проходят через gateway с аутентификацией/авторизацией  → отправляются в API внешних систем.мЧувствительные данные (шаблоны биометрии, сканы номеров) передаются только как в зашифрованном виде, без хранения на серверах приложения и прямой привязки к пользователю.
